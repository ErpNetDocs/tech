# Getting Started

This section provides the shortest path to a working Client Center.

It brings together the original Client Center setup guides into a first-time setup sequence: create the Client Center website, set up administrator access for managing users, and then add the first external user.

Use these pages when you are preparing a new Client Center for a customer or when you need to enable the first external users to sign in and work with it.

For detailed setting behavior, see [Configuration](../configuration/index.md).  
For background on roles and login behavior, see [Access model and external roles](../concepts/access-model-and-external-roles.md) and [Multi-customer login](../concepts/multi-customer-login.md).

## In this section

- [Set up a Client Center](set-up-a-client-center.md)  
  Create a website of type **Client Center**, assign it to the relevant enterprise company, and apply the initial website settings.

- [Set up administrator access](set-up-administrator-access.md)  
  Explain how administrator-level access is used when managing Client Center users and permissions.

- [Add your first external user](add-your-first-external-user.md)  
  Add a user after they create a local account, assign their access, and make the Client Center available to them.

## Recommended reading order

If you are setting up a Client Center for the first time, read the pages in this order:

1. [Set up a Client Center](set-up-a-client-center.md)
2. [Set up administrator access](set-up-administrator-access.md)
3. [Add your first external user](add-your-first-external-user.md)
