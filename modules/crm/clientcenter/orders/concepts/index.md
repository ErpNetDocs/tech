# Concepts

Use the pages in this section to understand how the **Orders** area in Client Center works.

These pages explain the difference between the list of existing orders and the **New Order** function, how order information is presented, and how order-related behavior is affected by access level and configuration.

Unlike the task-oriented guides in [Operations](../operations/index.md), the pages here focus on the logic and behavior of the Orders section.

## In this section

- [Orders page](orders-page.md)  
  Explains how existing orders are shown in Client Center, what users can see in the order list and document view, and how attached files appear in an order.

- [New Order behavior](new-order-behavior.md)  
  Explains how **New Order** works, including tabs, product selection, notes, promotional packages, availability behavior, and order placement behavior.

## Recommended reading order

If you are new to the Orders section, read the pages in this order:

1. [Orders page](orders-page.md)
2. [New Order behavior](new-order-behavior.md)

This sequence starts with the existing order documents shown in Client Center and then continues with the behavior of creating new orders.
