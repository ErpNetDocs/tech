# Concepts

Use the pages in this section to understand how the **User Management** area in Client Center works.

These pages explain how external users are assigned to customers, how customer-specific access is maintained, and how roles and restrictions determine what each user can see and do.

Unlike the task-oriented guidance in [Operations](../operations/index.md), the pages here focus on the logic and behavior behind user assignments in Client Center.

## In this section

- [User access and customer assignment](user-access-and-customer-assignment.md)  
  Explains how Client Center assigns access per customer through the **External Access** model and how roles, customer assignments, and **Days Back Access** work together.

## Recommended reading order

If you are new to the User Management section, read the pages in this order:

1. [User access and customer assignment](user-access-and-customer-assignment.md)

This page provides the main concept needed to understand how user administration works in Client Center.
