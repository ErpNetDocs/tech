# Concepts

The **Concepts** section explains how Client Center works and why it behaves as it does.

Use this section when you want to understand the access model, the relationship between users and customers, the navigation logic, the shared behavior of list pages, and the rules that determine whether files are visible to external users.

Unlike the step-by-step guides in [Getting Started](../getting-started/index.md) or the setting descriptions in [Configuration](../configuration/index.md), the pages here focus on the underlying logic and behavior of Client Center.

## In this section

- [Access model and external roles](access-model-and-external-roles.md)  
  Explains how Client Center access is assigned per customer through the **External Access** table and how external roles determine what users can see and do.

- [Multi-customer login](multi-customer-login.md)  
  Explains how one external user can access multiple customer accounts with the same sign-in and how Client Center switches between customer contexts.

- [Home and navigation](home-and-navigation.md)  
  Describes the Home page as the starting point in Client Center and explains how navigation depends on the user's role and the selected customer.

- [Shared grid behavior](shared-grid-behavior.md)  
  Explains the common interaction model used in Client Center list pages such as Orders, Invoices, Due Payments, and Payment History.

- [File visibility and downloads](file-visibility-and-downloads.md)  
  Explains where files appear in Client Center and how file visibility depends on both user role and file sharing through **Access Permissions**.

## Recommended reading order

If you are new to the Client Center concepts, read the pages in this order:

1. [Access model and external roles](access-model-and-external-roles.md)
2. [Multi-customer login](multi-customer-login.md)
3. [Home and navigation](home-and-navigation.md)
4. [Shared grid behavior](shared-grid-behavior.md)
5. [File visibility and downloads](file-visibility-and-downloads.md)

This reading order starts with the access model, then moves to customer context, page availability, shared interface behavior, and file-related visibility rules.
