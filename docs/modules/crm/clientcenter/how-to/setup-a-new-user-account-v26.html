How to define a user of a Client Center (v.26)

This is a step-by-step guide on defining a new user and configuring their access to a Client Center (CC) in ERP.net version 26.

For details on how to define a user of a Client Center in ERP.net version 24, please see this guide.

Note

If you haven't created a CC website already, please read Define a new Client Center.

Local account creation

Before a user can be granted access to a Client Center by an Administrator, they must first create a local account.

Accessing the CC log-in page allows them to open local account creation form. Required fields are Ð•-mail, Full Name, and Password.

Adding local user account in CC

Once a user has created their account, an Administrator of a Client Center must add them in the respective desired environment through the User Management page.

Administrators are assigned the external access role Admin, which guarantees them full reach of all customer data, including the ability to manage users.

Omission of this step will result in a critical lack-of-access error message for the new user when attempting to log in.

Step-by-step process

  1. If using multiple customers, make sure you are logged into the customer whose data you wish to share with the user.

  2. Navigate to the User Management page and click Add User.

    This will trigger a window where you need to provide the user's exact email address.

  3. Proceed to fill out the Days Back Access field. This is the maximum number of days the user is allowed to view data in the past.

  4. Finally, choose the external access role the user will be assigned.

    Users granted an external access role Admin or Owner are capable of adding other users to the CC environment(s) they are a part of.

  5. The new user can now log in with their credentials and immediately start using the Client Center.

Tip

To expand a user's access to more of your customers, switch to another customer profile and repeat all steps described above.
If you've passed all the steps successfully, you may proceed to apply global platform settings.

Warning

If any of the configuration settings are not properly applied, a respective error message will be shown.
